# DashCat

クリップボード履歴、システム監視、スリープ防止、マウスホイール反転を 1 匹の走る猫にまとめた軽量 macOS メニューバーアプリです。

macOS のメニューバーで、システムモニター、クリップボード管理（Maccy）、スリープ防止（Caffeine）、外付けマウスホイール方向の調整ツールを使っていました。複数のアイコン、複数のバックグラウンドプロセス——なんだか無駄に感じていました。そこでゼロから作り直し、本当に必要な機能だけに絞りました：システムモニタリング、クリップボード管理、スリープ防止、マウスホイール反転。モニターは Apple Silicon に最適化、クリップボード管理はすっきり効率的、スリープ防止とホイール反転は内蔵。余計なものは一切なし。

そうして DashCat が生まれました。メニューバーに座る 1 匹の猫——走る速度が速いほどシステム負荷が高いことを示し、左クリックでクリップボード履歴を検索でき、右クリックでスリープ防止、マウスホイール方向、モニターモード、言語切替を一覧できます。1 つのアイコンで複数の日常ツールの仕事をこなします。依存関係ゼロ、極低リソース、データはすべてローカルに保存。

## 機能

- **クリップボード管理**
  - 猫アイコンを左クリックでクリップボードパネルを表示
  - 検索ボックスでリアルタイムフィルタリング
  - クリックでコピー、`Option + Enter` でプレーンテキストとしてコピー
  - 項目を右クリックして上部にピン留め
  - テキストと画像に対応（画像保存はトグル可能、JPEG 圧縮）
  - 保持期間をカスタマイズ：7 / 14 / 30 / 90 日、無期限、または 1-365 日の任意の日数
  - すべてのデータはローカル保存——完全オフライン、データ収集なし

- **システムモニター**
  - デフォルトはコンパクト数値：CPU + メモリを C/M の 2 行パーセンテージで表示し、メニューバー幅を節約
  - カスタム表示では監視ソース（複合、CPU、メモリ）と表示方式（アニメーション、アニメーション + 数値）を選択可能
  - 猫のアニメーション速度がリアルタイムのシステム負荷を反映——速く走るほど負荷が高い
  - 複合モードでは CPU とメモリのうち圧力が高い方でアニメーションを駆動

- **コンパクトなバッテリー表示**
  - 猫アイコンとは別に表示できる、独立したメニューバー用バッテリー表示
  - パーセント記号を省いた細い数字に、淡い青の残量背景を重ねて表示
  - バッテリー駆動時は 20% 以下で数字/枠線がオレンジ、10% 以下で赤に変化
  - 電源接続中は自動で非表示にでき、メニューバーの幅を使わない
  - システムの電源情報を低頻度で更新し、アニメーションなしで軽量

- **スリープ防止**
  - デフォルト色：通常——システムはスリープ可能
  - 青色：システムのアイドルスリープを防止（ディスプレイはオフ可能）
  - オレンジ色：ディスプレイスリープを防止
  - 右クリックメニューから直接切替——猫の色がリアルタイムに変化

- **その他**
  - 11 言語対応：English、中文、日本語、한국어、Deutsch、Français、Español、Português、Italiano、繁體中文、Русский
  - 外付けマウスホイールの方向を反転し、トラックパッドは macOS の自然なスクロールのまま維持
  - Finderで TXT または Markdown ファイルを作成。作成前に保存先を表示し、別フォルダも選択可能
  - ログイン時起動に対応
  - 省電力設計：アニメーション上限 12fps、サンプリング間隔 5 秒、システムスリープ時に自動一時停止
  - 外部依存ゼロ——純粋な AppKit + Swift

## 動作環境

- macOS 13（Ventura）以降
- Apple Silicon Mac（M シリーズチップ）

## インストール

**方法 1：DMG インストーラー**

1. このリポジトリの [Releases](../../releases) ページから最新の `DashCat-<version>.dmg` をダウンロード
2. DMG を開き、DashCat をアプリケーションフォルダにドラッグ
3. 初回起動時、macOS が「アプリが破損している」または「開発元を確認できない」と表示する場合があります——これは未署名アプリに対する Gatekeeper の正常な動作です。ターミナルで以下を実行して検疫フラグを削除してください：
   ```bash
   xattr -cr /Applications/DashCat.app
   ```
   その後、ダブルクリックで通常通り起動できます。または、右クリック → 開く → ダイアログで「開く」をクリック。

**方法 2：ソースからビルド（Gatekeeper の回避不要）**

1. このリポジトリをクローン
2. Xcode で `DashCat.xcodeproj` を開く
3. **Signing & Capabilities** で自分の開発者アカウントを選択
4. `⌘R` で実行——Xcode が自動的に署名します

## 使い方

- **左クリック**猫アイコン：クリップボード履歴パネルを開く
  - 検索ボックスに入力してフィルタリング
  - 項目をクリックしてコピー
  - `Option + Enter` でプレーンテキストとしてコピー
  - 項目を右クリックしてピン留め/解除
- **右クリック**猫アイコン：設定メニューを開く
  - Monitor でコンパクト数値とカスタムアニメーション表示を切替
  - クリップボード設定で画像保存、保持期間、履歴クリアを管理
  - コンパクトなバッテリー表示と電源接続中の非表示を切替
  - Finder で新規ファイル作成、マウスホイール反転、言語切替、ログイン時起動の設定

## よくある質問

**クリップボードデータはどこに保存されますか？**

`~/Library/Application Support/DashCat/` — `clipboard.db` にテキストレコード、`Images/` に画像ファイル。履歴をクリアすると両方が消去されます。

**画像の保存にどれくらい容量を使いますか？**

画像は JPEG 圧縮で保存され、1 枚あたり数百 KB です。画像保存はデフォルトでオフです。有効にした場合、合計 500 MB の上限があり、上限を超えると最も古いピン留めされていない画像が自動削除されます。

**猫のアイコンの色は何を意味しますか？**

デフォルト → 通常のスリープ動作。**青色** → システムスリープを防止中。**オレンジ色** → ディスプレイスリープを防止中。右クリックメニューから切替できます。

**マウスホイール反転にアクセシビリティ権限が必要なのはなぜですか？**

DashCat はシステムイベント内のマウスホイールイベントを識別し、方向を反転する必要があるため、macOS のアクセシビリティ権限が必要です。未許可でもクリップボード履歴、システム監視、スリープ防止は動作します。右クリックメニューに案内とシステム設定への入口が表示されます。

**Finder ファイル作成で Finder の制御許可が必要なのはなぜですか？**

DashCat は「Finderで新規ファイル」を選んだ時だけ Finder の現在のフォルダを読み取ります。macOS が自動化権限を求める場合がありますが、DashCat がバックグラウンドで Finder を監視することはありません。このコマンドは DashCat のメニューにあり、Finder の空白部分のコンテキストメニューには追加されません。

**Intel Mac に対応していますか？**

いいえ。arm64 ビルドのみで、Apple Silicon 専用です。

**Maccy / CopyClip / Amphetamine とどう違いますか？**

DashCat はクリップボード管理（Maccy のような）、システムモニター、スリープ防止（Amphetamine / Caffeine のような）を 1 つの軽量メニューバーアプリに統合しています——1 つのアイコン、1 つのプロセス、依存関係ゼロ。メモリフットプリントを最小にするため純粋な AppKit で実装されています。

**初回起動時に「アプリが破損している」と表示されるのはなぜですか？**

プリビルド版は Apple Developer 証明書で署名されていないため、Gatekeeper がこのメッセージを表示します——アプリ自体は問題ありません。ターミナルで `xattr -cr /Applications/DashCat.app` を実行して検疫フラグを削除し、通常通り起動してください。この手順を避けたい場合は、ソースからビルドして自分のアカウントで署名してください。

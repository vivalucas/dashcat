# 功能设计

## 模块概览

| 模块 | 说明 | 负责人 |
|------|------|--------|
| 菜单栏主控 | 管理猫咪状态项、右键菜单、语言、开机启动和更新 | AppDelegate |
| 系统监控 | CPU / 内存采样、动画帧驱动、显示模式切换 | SystemMonitor + AppDelegate |
| 剪贴板 | 剪贴板轮询、SQLite 存储、搜索、Pin、图片管理 | ClipboardManager + ClipboardPanel |
| 防休眠 | IOPMAssertion 管理，三态切换 | AppDelegate |
| 电量 | 独立电量状态项、详情菜单、供电监听 | AppDelegate + BatteryStatusView |
| 滚轮反转 | 鼠标滚轮事件拦截和方向翻转 | ScrollManager |
| Finder 新建文件 | 读取 Finder 当前窗口目录并创建 TXT / Markdown | AppDelegate |

## 菜单栏主控

### 功能列表

- 左键打开剪贴板面板
- 右键打开设置菜单
- 维护语言、本地化标题和菜单状态
- 处理开机启动、更新检查、GitHub、联系入口

### 业务逻辑

`statusItem` 是主入口。点击事件根据左右键分流，右键展开菜单，左键显示或关闭 `ClipboardPanel`。

### 边界情况

- 菜单关闭后恢复左键行为
- 当前没有 `ClipboardPanel` 时再创建
- 语言切换后同时刷新菜单和弹窗本地化

## 剪贴板

### 功能列表

- 1 秒轮询 `NSPasteboard.general`
- 通过 `changeCount` 判断是否有变化
- 文本和图片写入 SQLite
- 搜索框实时过滤历史
- 支持 Pin、清除历史、保留天数、过滤词、图片保存
- 清除历史时提供保留固定项和清除全部两个选项

### 业务逻辑

`changeCount` 变化后读取当前剪贴板，先判断文本过滤词，再判断重复内容，最后写入数据库或保存图片。点击历史条目只负责回写剪贴板，不模拟粘贴。

### 边界情况

- 空内容跳过
- 重复内容跳过
- 过滤词命中时不写入历史
- 图片保存失败时不影响文本逻辑

## 系统监控与显示

### 功能列表

- 5 秒采样 CPU / 内存
- 在 `Compact Values` 下显示双行紧凑数值
- 在 `Custom Display` 下切换来源和显示方式
- 根据负载切换猫咪动画帧和颜色

### 业务逻辑

`MonitorMode` 决定看 CPU、内存还是综合值，`DisplayMode` 决定是否显示动画和数值。`cpuMemory` 模式直接固定为紧凑数值，不再开放外观配置。

### 边界情况

- 系统负载读取失败时回退默认值
- 普通模式和双数值模式使用不同渲染路径
- 菜单打开时的状态行采样不影响主循环

## 防休眠

### 功能列表

- 三态切换：关闭、阻止系统休眠、阻止屏幕休眠
- 通过 IOPMAssertion 管理系统休眠锁
- 颜色直接反映当前状态

### 业务逻辑

选择状态时先释放旧 assertion，再按新模式创建新的 assertion。切换结果同步保存到 `UserDefaults`。

### 边界情况

- 如果 assertion 创建失败，状态回退为关闭
- 系统睡眠 / 唤醒时暂停和恢复定时器

## 电量

### 功能列表

- 可选独立状态项
- 右键打开轻量详情菜单
- 左键循环切换防休眠状态
- 接电时可隐藏

### 业务逻辑

使用 IOKit Power Sources 读取电量、供电和充电状态。状态项本体只显示窄数字，未接电正常电量显示蓝色，接电 / 充电时通过粉色底纹和描边表达，不增加字符宽度。

### 边界情况

- 没电池或读取失败时移除状态项
- 接电隐藏开启时不占菜单栏空间
- 详情菜单只读，不提供高能耗 App 列表或强制充满动作

## 滚轮反转

### 功能列表

- 仅处理传统鼠标滚轮
- 触控板和连续滚动设备直接透传
- 辅助功能权限不足时提示并引导系统设置

### 业务逻辑

通过 `CGEventTap` 拦截 `.scrollWheel`，用 `scrollWheelEventIsContinuous` 区分设备类型，再对 delta 取反。

### 边界情况

- 用户关闭功能后立即停止 event tap
- 权限不足时不会阻塞其他功能

## Finder 新建文件

### 功能列表

- 从 DashCat 菜单触发
- 读取 Finder 最前窗口目录
- 支持 TXT / Markdown
- 支持创建前输入文件名
- 创建后用 Finder reveal

### 业务逻辑

先通过 AppleScript 取目标目录，再让用户在确认框里选择文件类型、输入文件名和可选文件夹，最后在目标目录创建空文件。

### 边界情况

- Finder 没有窗口时回退到桌面
- 自动化权限不足时给出失败提示
- 重名时自动递增文件名
